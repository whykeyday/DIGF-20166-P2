# 更新2.md（2026-02-28）— 交互测试：为什么一直蓝色呼吸灯？

## 现象（What happened）
- NeoPixel 一直显示 **蓝色呼吸灯**（CALM 模式），即使我尝试按压压力传感器 **3 次**，灯效仍没有进入其他模式。

## 说明（What it means）
- 这通常表示系统一直停留在 **Mode 1: CALM**，没有满足切换到 Mode 2/3/4 的触发条件。
- 我的代码里，Mode 2 触发条件是：在 **10 秒内 pickCount ≥ 3**（"抠袖子/按压事件"计数达到阈值）。
- 重要：`pickCount` 的计数方式是 **上升沿触发（edge detect）**：
  - 必须是 **按下 → 松开 → 再按下** 才能累积多次
  - 如果按住不放，或松开后读数没有回落到复位阈值以下，就不会累计到 3 次。

## 可能原因（Likely causes）
### 1) 计数没有真的达到 3（最常见）
- 我虽然"按了 3 次"，但可能：
  - 没有完全松开，传感器值没有回到复位阈值以下
  - 衣服缝制/拉扯导致传感器一直有残余压力
  - 平滑（EMA smoothing）让回落变慢，导致 `pickActive` 没有复位

### 2) 阈值/回差不适合当前传感器
- 当前参数（已调低）：
  - `PRESSURE_THRESHOLD_LOW = 200`（原 300）
  - `PRESSURE_HYSTERESIS = 15`（原 30）
  - 复位条件：`smoothPressure < 185`
- 如果传感器在"松开"时仍高于 185，则不会复位，也就无法多次计数。

### 3) 模式切换不明显（但仍需排除）
- 即使进入 Mode 2，若亮度很低或颜色接近，可能看起来仍像"蓝色呼吸"。
- 已通过 `DEBUG_MODE = true` 用强制颜色解决：
  - CALM = 暗蓝 | BREATH = 亮红 | OVERSTIM = 品红 | MOVE = 亮绿

## 下一步测试计划（Next steps / Debug plan）
### A. 用串口确认 pickCount 和 mode
- 打开 Serial Monitor（9600 baud），观察输出列：
  - `P_raw  P_smo  S_raw  S_smo  Picks  Mode  Seden  pickAct`
- 目标：确认是否真的出现 `pickCount >= 3`，以及 `Mode` 是否从 `0` 变为 `1`。
- 关注 `pickAct`：按下时应为 `1`，松开时应回到 `0`，如果一直是 `1` 说明没复位。

### B. 记录真实"抠袖子"动作数据
- 做真实佩戴状态下的"抠袖子"动作测量
- 记录 1 分钟数据，重新确定 baseline 与触发区间
- 最终用数据来定阈值，而不是凭感觉

### C. 调试完毕后恢复正式参数
- 确认逻辑正确后，将 `DEBUG_MODE` 设为 `false`
